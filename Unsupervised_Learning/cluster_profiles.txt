----------------------------------------------------------------
1. CLUSTERING EVALUATION SUMMARY
----------------------------------------------------------------
The optimal number of clusters was determined to be k=3 based on
the elbow method (WCSS scan) and silhouette analysis.

KMeans (k=3) was selected as the final model based on:
  - Lower WCSS          : 39,240.40  (vs Agglomerative: 41,793.36)
  - Better Silhouette   : 0.1357     (vs Agglomerative: 0.1062)
  - Better Davies-Bouldin: 2.1824    (vs Agglomerative: 2.3278)

Note: Low silhouette scores across all models indicate that the
clusters overlap in feature space — which is expected for
financial data where income and asset levels form a continuum
rather than clearly separated groups.

6. INTEGRATION STRATEGY
----------------------------------------------------------------
The cluster labels generated by KMeans (k=3) are integrated into
the loan approval advice system in the following ways:

  1. Personalized Risk Assessment:
     Each applicant is assigned to a cluster at prediction time
     using their financial features. The cluster profile is used
     to generate a personalized explanation of their approval
     likelihood beyond just the binary prediction.

  2. Targeted Advice Generation (Generative AI Integration):
     Cluster membership is passed as context to the Generative AI
     prompt templates, enabling the model to tailor advice to the
     applicant's financial tier:
       - Cluster 0 → Emphasize maintaining credit score
       - Cluster 1 → Emphasize improving credit and reducing debt
       - Cluster 2 → Emphasize managing loan-to-income ratio

  3. Enhanced Supervised Learning Features:
     The cluster label (0, 1, 2) is added as an additional feature
     to the supervised classification models (see:
     preprocessed_loan_data_v2_with_clusters.csv), providing
     group-level context that improves model interpretability.

----------------------------------------------------------------
7. LIMITATIONS
----------------------------------------------------------------
  - Low silhouette scores (0.1357 for k=3) indicate that cluster
    boundaries are soft — applicants near cluster boundaries
    could reasonably belong to adjacent clusters.
  - Clustering was performed on scaled features without the
    target label (loan_status), so clusters reflect financial
    profiles, not approval outcomes directly.
  - The dataset is balanced across three income tiers with
    similar approval rates (~61-64%), suggesting that no single
    financial tier has a decisive advantage in loan approval.
